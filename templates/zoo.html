Reference Samples

Annotated collection of public educational samples — other people's work studied for technique comparison. Each entry notes what it does well and what to look for in the code.

Lab use only. Every sample listed here is from a public repository and is linked for educational and analytical purposes. Run them only in an isolated virtual machine with no internet access and no connection to anything you care about. Detonating unknown binaries on your host machine or a shared network is not a risk worth taking.

Sliver — Open Source C2 Framework
Full implant framework written in Go. Covers everything in this project plus much more — mutual TLS, DNS beaconing, process injection, multiplayer operator support. Study it to see how production-grade Go C2 is structured, not to copy it wholesale.

Go-Inject — Shellcode Loader
Minimal Go shellcode injection using Windows API calls via the syscall package — the alternative to shelling out to cmd.exe that's mentioned in the OPSEC section. Good for understanding how to avoid spawning child processes entirely.

garble — Go Binary Obfuscation
A Go build tool that randomises symbol names, shuffles code layout, and applies literal obfuscation at compile time. Drop-in replacement for the go build command. The obfuscation phase of this project demonstrates what garble protects against.

pupy — Cross-Platform RAT
Python RAT with a rich feature set — memory-only execution, in-memory module loading, multiple transports. The Python equivalent of what this project builds in Go. Useful for comparing how the same concepts (persistence, exfil, C2) are implemented in a higher-level language.

Simple Python Keylogger
Minimal keylogger using the pynput library — demonstrates credential harvesting (T1056) in under 30 lines. Useful as a contrast to the reconnaissance approach: passive collection vs active command execution. Compare detection signatures.

PowerShell/Python post-exploitation framework. Heavily documented, built with detection evasion in mind from the start. Good for understanding how obfuscation, encrypted C2, and OPSEC are handled at scale — everything this project is building toward.

Mirai — IoT Botnet Source
Source code of the 2016 Mirai botnet, leaked and widely studied. Written in C. Demonstrates credential scanning, DDoS modules, and C2 architecture at industrial scale. Study the structure — persistence, scanning, C2 protocol — and compare to the Go approach.

A bare-minimum reverse shell in C — socket connection, fork, exec. Under 50 lines. Good for understanding what's happening at the syscall level when Go's net/http makes a connection. The Go version abstracts all of this; C shows the seams.

JavaReverse Eng.Free
Tool
Ghidra — NSA Reverse Engineering Suite
Free, open-source reverse engineering framework from the NSA. Load any of the compiled .exe binaries from this project and explore what the binary looks like before and after stripping. The obfuscation lesson specifically uses Ghidra for before/after comparison.
— ghidra-sre.org ↗
WindowsForensicsFree
Tool
Sysinternals Suite — Process Monitor, Autoruns, strings.exe
The Windows forensics toolkit. Autoruns catches the registry persistence from v3, strings.exe reveals hardcoded strings in the binary (showing why obfuscation matters), and Process Monitor traces every file, registry, and network event. All exercises in the lessons reference these.
— Microsoft ↗
WindowsLoggingFree
Tool
Sysmon — System Monitor
Microsoft's process, network, and file logging driver. Every detection table in this project references Sysmon event IDs. Install it on your lab Windows VM with a sensible config (SwiftOnSecurity's config is a good starting point) and watch your own payload get logged.
— Microsoft ↗
How to use these: Don't just run the samples and look at the output. Read the source code alongside your own v3 payload and compare how the same problems are solved differently. Mirai's C2 protocol is more complex than ours but solves the same problem — how does it identify beacons? How does it handle multiple targets? How does Empire's PowerShell obfuscation differ from XOR encoding? The interesting questions are comparative, not just "what does this do."
